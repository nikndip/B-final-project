# B-final-project

## Выпускная квалификационная работа (бакалавриат)

Репозиторий содержит исходный код выпускной квалификационной работы бакалаврского уровня, посвященной разработке веб-приложения для сопровождения сотрудников в рамках модуля питания корпоративной системы реабилитации.

## Назначение проекта

Разработанное программное средство предназначено для автоматизации процессов, связанных с персонализированным планированием питания, контролем пользовательской активности, обработкой заявок на поощрения, сопровождением обращений в поддержку и администрированием соответствующего функционального модуля.

Система реализована в виде серверного веб-приложения с ролевой моделью доступа и централизованным хранением данных в PostgreSQL.

## Цель разработки

Целью разработки является создание прикладной информационной системы, обеспечивающей:

- персонализированное взаимодействие сотрудников с модулем питания;
- разграничение полномочий между сотрудником, руководителем и администратором;
- хранение и обработку данных о рационе, достижениях, баллах, обращениях и служебных действиях;
- поддержку эксплуатационных сценариев, характерных для корпоративной среды.

## Функциональные возможности

### Функции сотрудника

- просмотр дашборда и персонального профиля;
- заполнение анкеты питания;
- работа с недельным планом питания;
- фиксация приемов пищи и контроля гидратации;
- просмотр достижений, рейтинга и накопленных баллов;
- подача заявок на получение поощрений;
- создание и сопровождение обращений в службу поддержки;
- восстановление доступа через подачу заявки администратору.

### Функции руководителя

- просмотр данных по сотрудникам своего подразделения;
- согласование или отклонение заявок на поощрения;
- начисление дополнительных баллов сотрудникам подразделения;
- работа с обращениями поддержки сотрудников своего подразделения;
- получение уведомлений о просроченных заявках и обращениях.

### Функции администратора

- управление пользователями модуля питания;
- назначение временного пароля при восстановлении доступа;
- просмотр анкет, карточек сотрудников и журнала баллов;
- управление каталогом достижений;
- обработка обращений поддержки;
- просмотр аудита действий;
- контроль SLA по заявкам и обращениям.

## Технологический стек

- Go 1.22
- PostgreSQL 14+
- `github.com/go-chi/chi/v5`
- `github.com/jackc/pgx/v5`
- HTML Templates (SSR)
- CSS
- Docker Compose для локального окружения базы данных

## Архитектурные особенности

- приложение построено по модульной структуре с выделением слоев конфигурации, доступа к данным, middleware, доменных моделей, HTTP-обработчиков и веб-представлений;
- аутентификация реализована через серверные сессии;
- для форм с изменением состояния добавлена CSRF-защита;
- схема базы данных инициализируется миграциями;
- предусмотрено наполнение демонстрационными данными для локального тестирования.

## Требования к окружению

Для локального запуска необходимы:

- Go 1.22 или выше;
- PostgreSQL 14 или выше;
- Docker и Docker Compose;
- Unix-подобная среда выполнения или совместимое окружение разработки.

## Порядок локального запуска

1. Перейти в каталог проекта:

```bash
cd <путь_к_каталогу_проекта>
```

2. Запустить контейнер PostgreSQL:

```bash
docker compose up -d
```

3. Создать локальный файл переменных окружения:

```bash
cp .env.example .env
```

4. При необходимости включить демонстрационные данные:

```env
SEED_DATA=true
```

5. Запустить приложение:

```bash
go run .
```

После запуска приложение будет доступно по адресу `http://localhost:8080`.

## Тестовые учетные записи

При значении `SEED_DATA=true` используются следующие демонстрационные учетные записи:

- сотрудник: `10001` / `passord1`
- сотрудник: `20001` / `password`
- руководитель: `30001` / `password`
- администратор: `90000` / `password`

## Переменные окружения

| Переменная | Значение по умолчанию | Назначение |
|---|---|---|
| `APP_ADDR` | `:8080` | адрес HTTP-сервера |
| `DATABASE_URL` | `postgres://rehab:rehab@localhost:5432/rehab_app?sslmode=disable` | строка подключения к PostgreSQL |
| `RUN_MIGRATIONS` | `true` | автоматический запуск миграций при старте |
| `SEED_DATA` | `false` | загрузка демонстрационных данных |
| `COOKIE_NAME` | `rehab_session` | имя cookie серверной сессии |
| `COOKIE_SECURE` | `false` | передача cookie только по HTTPS |
| `SESSION_TTL` | `168h` | время жизни пользовательской сессии |

## Миграции и данные

- миграции располагаются в каталоге `migrations/`;
- применение миграций выполняется в лексикографическом порядке;
- основной файл миграции: `001_init.sql`;
- текущая миграция содержит только безопасные additive-изменения схемы;
- демонстрационные данные формируются средствами `internal/db/seed.go`.

## Структура проекта

```text
main.go
cmd/
  complexity-report/  # утилита расчета параметров сложности
internal/
  config/             # конфигурация приложения
  db/                 # подключение к БД, миграции, демонстрационные данные
  middleware/         # сессии, логирование, recovery, CSRF, контекст
  models/             # доменные модели
  site/               # HTTP-обработчики и бизнес-логика
  web/
    static/           # статические ресурсы
    templates/        # HTML-шаблоны серверного рендеринга
migrations/           # SQL-миграции
docker-compose.yml    # локальное окружение PostgreSQL
```

## Практические замечания по эксплуатации

- в промышленной среде необходимо установить `COOKIE_SECURE=true`;
- рекомендуется использовать HTTPS на уровне reverse proxy или ingress;
- для промышленной эксплуатации следует применять отдельную базу данных и отдельные учетные данные;
- создание учетных записей должно выполняться только через административный контур;
- необходимо предусмотреть резервное копирование базы данных и мониторинг ключевых эксплуатационных событий.
